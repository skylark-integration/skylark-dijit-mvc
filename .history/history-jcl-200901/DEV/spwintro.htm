WebとHTMLはもともと、情報の表示や情報間の複雑な関連を管理するために開発された技術で、 Web は導入がとても容易であったため、多くの企業がアプリケーションのインターフェイスの構築手段として Webシステム を使い始めました。しかし大半のアプリケーションでは、ユーザビリティが以前より大幅に低下する結果となりました。その原因は、 HTML におけるサービス提供のメカニズムや UI コントロールの貧弱さ、そしてクライアント側データモデルの欠如にあります。

ただし、近年からWEBシステムへのニーズは高度化、多様化し、より複雑な機能、よりリッチなユーザーインタフェース・操作性や高いパフォーマンスなどの要望が高まっています。

Sparrow Framework はこのニーズに応じて、デスクトップアプリケーションと同等または以上のリッチなユーザーインタフェースを持てる、Web 2.0アプリケーションを効率的なプログラミングモデルで構築できるフレームワーク及び開発手法を提供しています。
