Javascriptの歴史

　　JavaScriptは 最初は Sun Microsystems社とNetscape Communications社が開発した、  Webブラウザなどでの利用に適したスクリプト言語で  ブラウザ以外のソフトウェアにも簡易な制御プログラムの記述用言語として移植されており、  Microsoft社のWindowsやWebサーバソフト「IIS」、Macromedia社の「Flash」などがあります。

　　各社の実装に微妙な違いがあり、ブラウザによって使えない機能があったり同じプログラムでも挙動が異なったりする問題があったため、  ヨーロッパの標準化団体ECMAがソフトウェアメーカーに呼びかけて、通称「ECMAScript」と  呼ばれる標準を発行しました。2006年現在ECMAScriptの最新バージョンは3 (3rd edition)で4rd editionの策定も進みつつあっています。

JOFとは

　　JOFはJavascript  Object Frameworkの略称でプロトタイプベースのJavascritpに基づいてクラスベースのオブジェクト指向モデル及びフレームワークを実現したものです。Javascritpは最初からサーバに再アクセスせずにクライアント側に動的にウェブページを変更できる技術として使われてきたスクリプト言語であり、これからRIAまたはAJAXアプリケーションの広がるにつれて、本格的な一プログラム言語として広がります。JOFを利用することで大型クライアントアプリケーションをもっと容易的に構築または管理することが出来ます。

 

 

１．動的なクラスベースオブジェクト指向

1.1 オブジェクト指向モデル

　

       Javascriptはプロトタイプベースのオブジェクト指向言語として広く知られています。 C+＋、Javaなど普通のクラスペース言語と違ってJavascript にはクラス・インスタンスなどの概念を持たずに全てのオブジェクトをスロットなどと呼ばれる、アクセス名と結びついた汎用の変数領域の塊でできた可変構造体として捉え直し、そこに他のオブジェクトへの参照やメソッドなどを自由に定義できるようにしています。つまりオブジェクトは参照とメソッドの集積体であり、いかなる静的構造とも結びつかない。クラスが存在しないJavascript世界ではテンプレート処理によるインスタンス化ができないため、新しいオブジェクトは完全に空の状態か、または他のオブジェクトの複製（クローン）によって作成される。プロトタイプベースでの継承は一般にこのクローンによる特性の引き継ぎといえます。クラス概念を持っていないのは大型アプリケーションの設計、整理するには難しいですがC+＋、Javaなど普通のクラスペース言語はクラスが静的な構造と結びついていることも時に問題になっています。

　　JOFはプロタイプベースのJavascritpにクラスベースのオブジェクト指向フレームワークを実現し、更にC+＋、Javaなど普通のクラスペース言語はクラスが静的な構造と結びついていることと違ってJOFのクラスは動的に作成したり、削除したり、変更したりすることが出来ます。

　　JOFにおいてはインスタンスを生成、参照する前にそのインスタンスが所属しているクラスを定義しておく必要があります。同じクラスの各インスタンスはクラスによって定義されたフィールド及びメッソドしかアクセスできないのでインスタンスは独自のフィールド及びメッソドを持たないです。

　　またJOFではネームスペースというものが存在しています。ネームスペースのなかにはクラスまたはサブネームスペースが入っていて、自分でもフィールドまたはメッソドを持つことが出来ます。ネームスペースも動的に作成したり、削除したり、変更したりすることが出来ます。

　　クラスもネームスペースもメッソドもオブジェクト扱いされていて、実際にはクラスとネームスペースはHashクラス（連想配列）のインスタンス、メッソドはMethodクラスのインスタンスとなっています。

1.2オブジェクトの分類

　　　JOFにてすべてのオブジェクトを以下のように分類することができます。。

①     クラス・オブジェクト

②     インスタンス・オブジェクト

③     ネームスペース・オブジェクト

④     メッソド・オブジェクト

　　　またJOF管理範囲外になっているjavascriptオブジェクト、ＤＯＭオブジェクト、ActiveXオブジェクトなどは未知オブジェクトと分類されます。

　　　オブジェクトのタイプを取得するにはSystem.objType(obj)メッソドを使えます。

　　　オブジェクトのタイプを識別するには以下の定数が定義されています。

System.OT_INSTANCE                    ：インスタンス

System.OT_NAMESPACE                ：ネームスペース

System.OT_CLASS                            ：クラス

System.OT_METHOD                       ：メッソド

System.OT_OT_EXTERNAL           ：未知

System.OT_UNDEFINE                    ：未定義

System.OT_NULL                              ：NULL

1.3オブジェクトのプロパティ

　　　オブジェクトはメンバープロパティを持つことができます。プロパティの種類を分類するとフィールドとメッソドと2種類があります。

　　　JavaScriptには他の言語と異なりメンバー変数を隠蔽する機能はありませんので隠蔽したいプロパティがある場合には名前の頭に"_"（アンダースコア）をつけるネーミングルールで、運用的に対応することとします。

２．クラス

2.1クラスの定義

　　　クラスを新しく定義するにはspw.System. defineClass()を使います。

           {Class} System.defineClass({Hash} cdef)

           cdefの構造は以下のようにです。

　　　　{

        name  : クラスのフール名称の値, 

      superc : クラスの親クラス,

      statics  : {

                F1  : フィールド１の値

　　　  F2  : フィールド2の値

　          M1 : function() {

                },

                M2 :function() {

                }

      },

      instances: {

                F1  : フィールド１の値

　　　  F2  : フィールド2の値

　          M1 : function() {

                },

                M2 :function() {

                }

      }

｝

staticsはクラスのスタティックプロパティを格納し、instancesはクラスのインスタンスプロパティを格納します。

 

同じ名称のクラスがすでに存在した場合、再定義することになります。

 

2.2クラスの変更

　　クラスオブジェクトはHashクラスの特殊なインスタンスでもあり、各クラスプロパティ及びメッソドは<name:value>の形でHashインスタンスに存在します。また各インスタンスプロパティ及びメッソドも<name:value>でHashインスタンスに纏めてinstannseというキーでクラスに対応するHashインスタンスに格納されています。

　　Hashインスタンスのキーを削除したり、追加したり、キーの値を変更したりすることによってクラスの定義を動的に変更することが出来ます。

2.3クラスの削除

　　　クラスを新しく定義するにはspw.System. deleteClass()を使います。詳しくは6節にご参照ください。

 

2.4クラスの継承

継承 (extends) とは、あるクラスの実装を土台にして、別のクラスを実装するための方法です。元になるクラスをスーパークラスと呼び、それを継承する新しいクラスをサブクラス（継承クラス）と呼びます。

一つのクラスは、任意の個数のサブクラスを持つことが出来ますが、逆に、スーパークラスは一つしか持てません。これを単一継承と呼び、クラスは継承されることで枝葉を広げていく木構造を成しています。そして、遡っていけば根っ子となる一つのスーパークラス（ルート・クラス）に行きつきます。

クラスの木構造の頂点、ルートとなるクラスが spw.Instance クラスです。スーパークラスを明示しないクラスの場合、自動的にこのクラスから派生されたことになります。スーパークラスを明示した場合でも、そのクラスも Instance クラスのサブクラスか、または Object クラスを継承したクラスのサブクラスのいずれかなので、最終的にはあらゆるクラスが Object クラスをスーパークラスに持つことに変わりはありません。

３．ネームスペース

　　ネームスペースの中にはクラスまたは他のサブネームスペースを格納することができます。またネームスペースはプロパティまたはメッソドを持つことができます。

3.1ネームスペースの定義

　　　ネームスペースを新しく定義するにはspw.System. defineNs()を使います。詳しくは6節にご参照ください。

           {Namespace} System.defineNs({Hash} ndef)

           ndefの構造は以下のようにです。

　　　　{

       name  : ネームスペースのフール名称の値, 

      props  : {

                F1  : フィールド１の値

　　　  F2  : フィールド2の値

　          M1 : function() {

                },

                M2 :function() {

                }

      }

｝

同じ名称のネームスペースがすでに存在した場合、再定義することになります。

3.2ネームスペースの変更

　　ネームスペースオブジェクトもクラスオブジェクトと同じように、Hashクラスの特殊なインスタンスでもあり、各プロパティ及びメッソドは<name:value>の形でHashインスタンスに存在します。

　　Hashインスタンスのキーを削除したり、追加したり、キーの値を変更したりすることによってクラスの定義を動的に変更することが出来ます。

 

3.3ネームスペースの削除

　　　ネームスペースを削除するにはspw.System. deleteNs()を使います。詳しくは6節にご参照ください。

3.4.1getNsName

　例えば

                var s = System.getNsName();

     sの値は”spw.System”になります。

 

４．メッソド

4.1メッソドの定義

　　メッソドを定義するにはfunctionキーワードを使えます。

                function  add(x,y) {

                                Return x+y;

                }

    またメッソドはMethodクラスのインスタンスですのでMethodクラスのコンストラクタを呼び出すことでメッソドを定義することもできます

                var method = new Method(“x”,”y”,”return x+y;”);

4.2 this

 　　メッソドの中におけるステートメントはthisキーワードを使って、カレントオブジェクトを参照することができます。一般に this はあるメソッド内でそのメソッドを呼び出したオブジェクトを参照しますがcall()、apply()でメッソドを実行する場合、パラメータとして明確に指定することもできます。

５．定義済みオブジェクト

JOFが幾つかのクラス及びネームスペースを定義しておいてます。これらのクラスまたはねー巣スペースオブジェクトは全てspwというネームスペースに格納されていますが呼び出す時にネームスペース名を省略することも可能です。
