１．特徴

　　Sparrow Frameworkはアプリケーションプレゼンテーションからビジネスロジックを分離して、以下の特徴を持っています。

n         オープンアーキテクチャーで高い柔軟性、拡張性を持つフレームワーク

Ø       Sparrow Frameworkを構成する各部分は互いに独立性を持っていますので、それぞれの部分を必要に応じて再組み合わせたり、一部分を入れ替えたり、または既存のシステムに一部分だけを導入したりすることができます。

Ø       イベント、リソースをインテリジェントに管理していますので、共通な機能を変更したり、追加したりすることが容易にできます。

Sparrow Frameworkはベースとしてプロジェクト向けのフレームワークにカスタマイズしやすいように設計されています。

n         データドリブンのプログラミングモデルをサポート　

　　ほぼ全てのソフトウェアアプリケーションがデータアクセス及びデータ処理をベースとするコードから構成されています。これは特にビジネスプロセス向けアプリケーションにおいでは最も顕著です。

　Sparrow Frameworkではアプリケーションデータモデルをセンタリングして、データモデルによって、プレゼンテーションとビジネスロジックを結びつけることができます。

　　プレゼンテーションとビジネスロジックは別々開発、単体テストすることができますので、開発メンバーの分業がもっと容易になります。また開発後のプログラムの仕様変更対応なども容易に行えます。

n         XMLベースで宣言的なモデルをサポート

　　開発者およびデザイナは XML ベースのモデルを通じて、ユーザーインターフェイスを宣言的に指定できます。

n         リッチインターネットアプリケーション

Sparrow Frameworkはリッチなクライアントアプリケーションの開発で必要とされる高度で洗練された GUI コントロールをアプリケーションプログラマに提供します。プログラマはこれらのビルトインコントロールのオブジェクトを作成して、いくつかのプロパティを割り当てるだけでアプリケーションの主要な GUI コンポーネントを作成することができます。また既存のコントロールを継承して独自のコントロールも簡単に作成できます。

Ø         標準的にサポートされているコントロール

コマンコントロール

ラベル、ボタン、テキストボックス、コンボボックス、ピッチャボックス、リストボックス、

グループボックス、ラジオグループ、ラジオボタン、インラインフレーム、プログレスバー、カレンダー、スライダー

メニューバー、メニューボックス、ポップアップメニュー

タブコントロール

グリッド

ツリービュー

ダイアログボックスとメッセージボックス

Ø         ドラッグ＆ドロップサポート

　　基底クラスでの共通実装ですべてのコントロールがドラッグ＆ドロップをサポートします。

Ø         高度な自動配置機能

すべてのコントロールは横または縦方向に位置をコンテナーにどのように自動的に結びつけるのかを設定できます。

　　　　すべてのコントロールはコンテナーにどの位置(左、右、上、下或いはクライアント領域全体)に占めるのかを設定できます。

　　　　Splitterコントロールを利用すれば実行時にマウスで移動することにより、同じ親コンテナーのそれぞれのコントロールのサイズを変更できます。

　　以上三つの機能を利用して、高度な画面レイアウトを簡単に構築することができます。

Ø         シンプルなプログラミングインタフェース

HTMLとCSSの高度なレンダリング機能を出来る限り継承しながら、複雑なHTMLとCSS仕様を隠蔽してシンプルなAPIを提供します。

n         セキュアなアプリケーション

n       TCOの削減

クライアントサイドでは、インストール、アップグレード、パッチの摘要といった作業が不要です。

非同期通信を利用してクライアントとサーバと差分だけでやりとりすることで、サーバ負荷を劇的に軽減することができます。
